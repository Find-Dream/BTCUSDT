# BTCAlpha - 基于 OKX BTCUSDT 永续合约的比特币智能量化交易系统

基于 OKX BTC/USDT 永续合约的智能量化交易系统。系统通过机器学习方法对历史盈利 K 线形态进行特征提取与记忆，在实盘中实现形态的自动识别与匹配，触发对应的开仓/平仓指令。

在信号过滤层面，系统融合 **12 个维度**的多源数据进行综合评分，涵盖：

- **技术面**：趋势方向（EMA 多空排列）、MACD 动量、K 线振幅与形态
- **成交量**：量价关系、突破有效性、量价背离
- **微观结构**：订单簿不平衡度（OBI）、主动买入比
- **衍生品数据**：资金费率、多空持仓比
- **宏观情绪**：市场波动率、社交媒体情绪、BTC 市值占比

多维度评分汇总后与动态阈值比较，仅在信号充分收敛时执行下单，从根本上规避单一指标失效的风险。

欢迎大家积极参与本项目，分享量化交易的先进经验和算法 → [经验分享](https://github.com/btcusdt-cc/BTCAlpha/issues)

## 安装与启动

### 1. 克隆并安装依赖

```bash
git clone https://github.com/btcusdt-ai/BTCAlpha.git
cd BTCAlpha
pip install -r requirements.txt
```

### 2. 配置 OKX API

拷贝配置文件

```
cp conf/okx_api_example.json conf/okx_api.json
```



编辑 `conf/okx_api.json`：

```json
{
  "api_key": "你的 API Key",
  "secret_key": "你的 Secret Key",
  "passphrase": "你的 Passphrase",
  "flag": "1"
}
```

> `flag: "1"` 为 OKX 模拟盘（推荐新手先用模拟盘测试），`flag: "0"` 为实盘。

### 3. 配置代理（如需要）

如果所在地区无法直连 OKX，在 `conf/config.yaml` 中配置代理：

```yaml
runtime:
  proxy: http://192.168.1.1:7890
```

### 4. 启动系统

```bash
python3 main.py
```

浏览器访问 `http://localhost:5000`

**其他启动方式：**

```bash
python3 main.py --no-scheduler   # 只启动 Web，不运行交易调度（纯查看模式）
```

## 首次使用流程

```
① 启动系统，打开浏览器进入首页
         ↓
② 在「K线图/回测」页面加载历史 K 线，找到历史上出现明显上涨/下跌的点位
         ↓
③ 点击图表上的蜡烛，选中该时间节点
         ↓
④ 点击「开多」或「开空」按钮，把该图形记录为交易模型（晋升）
         ↓
⑤ 多晋升几个不同时间段的典型图形，建立模型库
         ↓
⑥ 在「参数调节」页面确认各项参数符合预期
         ↓
⑦ 确认使用模拟盘，观察系统是否正常命中信号并模拟下单
         ↓
⑧ 满意后，按照「交易安全说明」开启实盘
```

### 参数调节
<img width="3408" height="1204" alt="image" src="https://github.com/user-attachments/assets/a62ecf08-eca5-43d1-8f99-ff8e92b0a980" />

用于调整系统运行参数，保存后**立即热更新**，无需重启系统。

页面分为四个区块，每个参数旁有说明文字：

#### ⚡ 交易设置

| 参数 | 建议 |
|------|------|
| 实盘下单 | 新手请保持关闭，先用模拟盘观察 |
| 确认口令 | 实盘时必须填写 `I_KNOW_THE_RISK` |
| 交易方向许可 | 0=全停 1=只做多 2=只做空 3=双向 |
| 杠杆倍数 | 建议从 1 倍开始，控制风险 |
| 基础仓位 | 每笔开仓的初始合约张数 |
| 加仓总开关 | 关闭后亏损时不再马丁加仓 |

#### 📐 策略阈值

| 参数 | 说明 |
|------|------|
| 开仓振幅门 | K 线振幅需大于此值才触发，值越大信号越少越强 |
| 多/空头止盈系数 | 盈利超过此比例才止盈。1.005 = +0.5% |
| 高低位缓冲比例 | 在当日最高/最低价附近的此比例内不开仓（避免追高杀低）|

#### 🔍 方向过滤器

对每笔开仓进行多维度评分（趋势 + 成交量），总分达到阈值才允许开仓：

| 参数 | 说明 |
|------|------|
| 实盘通过阈值 | 实盘4个维度评分，满分±14，建议6~8 |
| 回测通过阈值 | 回测仅2个维度，满分±8，建议3~5 |
| 加仓也过滤 | 开启后马丁加仓前也需通过方向评分 |

#### 😱 市场情绪指数过滤

使用市场情绪指数（0-100）限制开仓方向：

| 分值区间 | 开仓限制 |
|---------|---------|
| ≤ 20 或 ≥ 80 | 停止所有开仓 |
| 20 < x < 40 | 禁止开多 |
| 40 ≤ x ≤ 60 | 不限制 |
| 60 < x < 80 | 禁止开空 |

此外，若近几日情绪持续上升，额外禁止开空；持续下降，额外禁止开多。

## 配置参数详解

配置文件位于 `conf/config.yaml`，可通过「参数调节」页面可视化修改。

### runtime（运行参数）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `inst_id` | BTC-USDT-SWAP | 交易标的（OKX 永续合约代码）|
| `bar` | 1m | 主匹配 K 线周期（1m/5m/15m）|
| `loop_interval` | 1 | 交易主循环间隔（秒）|
| `proxy` | 空 | HTTP 代理，无法直连 OKX 时填写 |
| `domain` | https://www.okx.com | OKX 接口域名 |

| `web_port` | 5000 | Web 服务端口 |

### trade（交易参数）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `live_enabled` | false | 是否允许向 OKX 发送订单 |
| `confirm_live` | — | 实盘确认口令，须填 `I_KNOW_THE_RISK` |
| `td_mode` | cross | 保证金模式（cross=全仓 / isolated=逐仓）|
| `lever` | 1 | 杠杆倍数 |
| `start_sz` | 3 | 每笔开仓初始张数 |
| `sz_divisor` | 15 | 仓位系数分母，实际张数≈可用量/此值 |
| `pos_ok` | 3 | 允许方向（0禁止 1只多 2只空 3双向）|
| `order_ttl` | 11 | 订单时效（秒），超时跳过 |

### strategy（策略阈值）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `amp_open` | 10 | 开仓振幅门（USDT），值越大信号越少 |
| `amp_close` | 20 | 平仓振幅门（USDT）|
| `take_profit_long` | 1.005 | 多头止盈系数（均价的1.005倍=+0.5%）|
| `take_profit_short` | 0.995 | 空头止盈系数（均价的0.995倍=-0.5%）|
| `add_pos_enabled` | false | 马丁加仓总开关 |
| `add_pos_max_times` | 3 | 最大加仓次数 |

| `day_range_buffer` | 0.05 | 日内高低位缓冲比例（5%）|
| `situation_extreme` | 1500 | 极端行情价差阈值 |
| `situation_1h_pct` | 0.01 | 1h 形势判定偏离比例（1%）|

| 参数 | 默认值 | 说明 |

## 模型库：系统的核心

模型库是系统的灵魂。

### 什么是"晋升"

在 K 线图上，找到一根你认为「这个走势出现时，价格之后确实涨了（或跌了）」的 K 线，点击它，选择操作类型（开多/开空/平多/平空），系统会把这根 K 线前25根的均线走势特征压缩成15个数字存入模型库。

### 系统如何使用模型

系统实时监控当前 K 线，每隔1秒计算一次当前图形的15个特征数字。如果与模型库中某条记录高度匹配，就认为「当前图形命中了该模型」，触发对应的交易决策。

### 如何建立好的模型库

1. **用回测找历史信号**：选一段历史时间运行回测，查看「建议晋升」列表——这里列出了有明显振幅但未被命中的点位
2. **按振幅优先**：建议晋升列表按振幅从大到小排序，优先晋升振幅大的
3. **定期清理失效模型**：回测后查看「失败模型ID」列表，删除命中后价格反向运动的模型
4. **不同市场环境多晋升**：趋势行情和震荡行情的图形不同，尽量覆盖多种形态

### 什么时候晋升

- 当你看到行情出现了一次明显的单边上涨或下跌，但系统没有开仓时
- 回测「建议晋升」列表中振幅大的点位
- 手动复盘时发现的高质量信号点

## 交易安全说明

系统默认处于**纯模拟模式**——只记日志，不向交易所发送任何请求。

开启实盘需要**三个条件同时满足**：

| 条件 | 位置 | 值 |
|------|------|-----|
| 允许下单 | `config.yaml → trade.live_enabled` | `true` |
| 确认口令 | `config.yaml → trade.confirm_live` | `I_KNOW_THE_RISK` |
| 实盘凭证 | `conf/okx_api.json → flag` | `"0"` |

任一条件不满足，系统自动回退为模拟成交。

**强烈建议**：
- 先在模拟盘（`flag: "1"`）完整运行至少一周，确认系统行为符合预期
- 首次实盘使用最小仓位（`start_sz: 1`，`lever: 1`）
- 关闭加仓功能（`add_pos_enabled: false`），观察单笔交易效果

## 常见问题

**Q: 系统启动后一直没有开仓信号？**  
A: 检查模型库是否为空（模型管理页），没有模型就没有信号。先在 K 线图页面晋升几个模型。

**Q: 所有信号都被方向过滤器拒绝（score 很低）？**  
A: 尝试降低 `direction_filter.threshold`（实盘）或 `backtest_threshold`（回测），默认值6和4适合大多数情况，行情平淡时可降至4和3。

**Q: K 线图显示空白？**  
A: 检查代理配置是否正确，`conf/config.yaml → runtime.proxy`。也可查看 `log/kline.log` 获取详细错误信息。

**Q: 如何完全停止系统？**  
A: `Ctrl+C` 停止进程。如果用的是模拟盘，不会产生任何实际影响。

**Q: 参数改了但交易行为没变？**  
A: 参数调节页的「保存」会立即更新 `config.yaml`，下一轮交易循环（约1秒后）自动生效，无需重启。
